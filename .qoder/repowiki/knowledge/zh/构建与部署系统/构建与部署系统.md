本项目为单二进制 Go 程序，采用极简的本地编译加 systemd 服务化部署模式，无 CI/CD、Dockerfile、Makefile 等自动化构建体系。

构建方式：使用 go build -o ipv6-proxy main.go 直接编译单个源文件，依赖通过 go.mod（当前为空）管理。安装脚本 scripts/install.sh 提供一键安装流程：克隆源码、go mod tidy、go build、复制配置文件、写入内核参数、注册 systemd 服务。

部署方式：通过 systemd/ipv6-proxy.service 将二进制作为系统服务运行，由 install.sh 自动复制到 /etc/systemd/system/ 并启用。运行时通过命令行 flag 配置监听端口和 IPv6 前缀，无需外部配置文件（尽管仓库提供了 config.yaml.example，但主程序未读取）。

关键约束：目标平台为 Linux（依赖 systemd、ndppd、sysctl），不支持跨平台构建。需要 root 权限执行安装脚本，以修改内核参数和注册系统服务。依赖 ndppd 进行 IPv6 NDP 代理，需手动配置路由表使本地可路由到分配的 /112 前缀。

缺失能力：无版本发布流程、无交叉编译、无容器化、无单元测试或集成测试脚本。无 GitHub Actions 或其他 CI 配置，代码提交后不会触发任何自动化检查。
