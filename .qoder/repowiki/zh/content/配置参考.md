## 简介
本配置参考面向 IPv6 代理池（基于 /64 或 /112 前缀的随机出口代理），提供命令行参数说明、默认值与使用示例，覆盖单机、集群与容器化部署的最佳实践；同时给出系统级配置要求（IPv6 路由、ndppd 代理、内核参数优化）以及性能调优建议与常见问题排查方法。

**重要更新**：生产环境推荐使用 /112 子网隔离标准，确保同一 /64 地址段下的多台服务器互不干扰，提高部署安全性和可管理性。

## 核心组件
- HTTP CONNECT 代理：监听指定地址，仅处理 CONNECT 方法，劫持连接后以 tcp6 拨号到目标，双向转发数据。
- SOCKS5 代理：实现握手、请求解析（支持域名与 IPv6）、tcp6 拨号与双向转发。
- 随机源 IP：根据 -prefix 指定的 IPv6 前缀范围，按主机位随机填充，确保每次出站使用不同源地址。
- 并发限流：通过内置信号量限制最大并发连接数，避免资源耗尽。
- 详细日志模式：通过 -v 标志启用详细的调试日志输出，便于问题诊断。

## 详细组件分析

### 命令行参数
- -http
  - 含义：HTTP CONNECT 代理监听地址
  - 默认值：0.0.0.0:53420
  - 示例：-http 0.0.0.0:53420
- -socks
  - 含义：SOCKS5 代理监听地址
  - 默认值：0.0.0.0:53421
  - 示例：-socks 0.0.0.0:53421
- -prefix
  - 含义：出站使用的 IPv6 前缀（CIDR），用于随机源地址分配
  - 默认值：240e:6b0:50::/112
  - 示例：-prefix 240e:6b0:50::/112
- -c
  - 含义：最大并发连接数（信号量上限）
  - 默认值：5000
  - 示例：-c 10000
- -v
  - 含义：启用详细日志/调试模式
  - 默认值：false（关闭）
  - 示例：-v
  - 说明：启用后会输出详细的调试信息，包括连接建立过程、错误详情等，有助于问题诊断

注意：
- 若 -http 为空字符串则不启动 HTTP 服务；-socks 同理。
- -prefix 必须为合法的 CIDR 表示，否则程序将退出并打印错误。
- -v 标志仅在需要调试问题时使用，生产环境建议关闭以避免日志过多影响性能。

**更新**：默认 IPv6 前缀已更新为 `240e:6b0:50::/112`，采用简化的 /112 子网格式，符合生产环境子网隔离标准。

### 系统级配置要求
- 内核参数（由安装脚本写入 sysctl.d）
  - net.ipv6.ip_nonlocal_bind = 1
  - net.ipv6.conf.all.forwarding = 1
  - net.ipv6.neigh.default.gc_thresh1 = 1024
  - net.ipv6.neigh.default.gc_thresh2 = 4096
  - net.ipv6.neigh.default.gc_thresh3 = 102400
  - net.ipv4.tcp_tw_reuse = 1
  - net.ipv4.ip_local_port_range = 1024 65535
  - net.ipv4.tcp_fin_timeout = 15
- 本地路由
  - 添加 local 路由指向本机接口，例如：ip route add local <your-prefix>/112 dev <interface>
- ndppd 代理
  - 安装并配置 proxy 规则，将指定网段静态代理到路由器，使 /112 下任意地址可达

**更新**：路由配置现已推荐使用 /112 子网格式，提供更好的子网隔离效果。

### 部署最佳实践

#### 单机部署
- 适用场景：开发测试或小规模使用
- 要点：
  - 启用 -http 和 -socks，绑定 0.0.0.0 或具体网卡 IP
  - 设置 -prefix 为可用 /112 子网（推荐使用默认的 240e:6b0:50::/112）
  - 调整 -c 匹配单机资源（如 5000~10000）
  - 配置内核参数与本地路由，启动 ndppd
  - 开发调试时可启用 -v 标志查看详细日志
- 参考命令（见 README 快速开始与手动部署）

**更新**：单机部署现推荐使用 /112 子网格式，简化配置并提高安全性。

#### 集群部署
- 适用场景：多节点横向扩展
- 要点：
  - 每个节点使用不同的 /112 前缀，避免冲突
  - 在负载均衡层（如 L4/L7）对多个节点做健康检查与权重分发
  - 统一内核参数与 ndppd 配置模板，保证一致性
  - 监控指标：并发连接数、错误率、延迟分布、端口复用情况
  - 生产环境禁用 -v 标志以减少日志开销
- 子网规划建议：
  - 服务器A：240e:6b0:50::/112
  - 服务器B：240e:6b0:50:2::/112
  - 服务器C：240e:6b0:50:4::/112
  - 以此类推，每两个 :: 递增一个 /112 子网

**更新**：集群部署现强制推荐使用 /112 子网隔离标准，确保各节点间完全隔离。

#### 容器化部署
- 适用场景：Kubernetes/Docker 环境

  - 每个 Pod/Container 使用独立的 /112 子网

### 并发与限流机制
- 信号量限流：-c 决定最大并发连接数，超出时返回"too many connections"
- 连接生命周期：握手成功后建立 tcp6 连接，双向 io.Copy 转发，完成后释放信号量
- 详细日志：在 -v 模式下会记录信号量获取状态和当前连接数

## 性能考虑
- 并发连接数
  - 合理设置 -c，结合 CPU、内存、文件描述符限制评估
  - 观察日志中的拒绝计数与后端错误率，动态调整
- 内存使用优化
  - 减少不必要的缓冲拷贝，保持 io.Copy 直转
  - 控制日志级别与频率，避免高吞吐下 IO 瓶颈
  - 生产环境禁用 -v 标志以减少日志输出开销
- 网络参数调优
  - 启用 net.ipv6.ip_nonlocal_bind 允许非本地地址绑定
  - 调整 neigh gc_thresh 提升邻居表容量
  - 调整 tcp_tw_reuse、ip_local_port_range、tcp_fin_timeout 缓解端口耗尽与 TIME_WAIT 压力
- 路由与 NDP
  - 确保 local 路由与 ndppd 规则生效，避免丢包与不可达
- 与聚合层配合
  - 与 v2ray/xray 的 smux 聚合可降低 conntrack 压力，提高吞吐
- 子网隔离优化
  - 使用 /112 子网可减少 ARP/NDP 表项数量，提升网络性能
  - 合理的子网划分有助于故障隔离和性能监控

## 故障排除指南
- 无法绑定端口
  - 检查 -http/-socks 是否被占用，确认防火墙放行
- 拨号失败（502/连接拒绝）
  - 确认 -prefix 合法且路由已添加（推荐使用 240e:6b0:50::/112）
  - 检查 ndppd 规则是否正确，目标是否可达
- 并发过高导致拒绝
  - 增大 -c，优化上游处理能力，必要时水平扩展
- 日志过多影响性能
  - 降低日志级别或采样记录，关注关键错误码
  - 生产环境禁用 -v 标志
- 证书问题（TLS 场景）
  - 确认证书 CN/SAN 与域名/IP 一致，私钥权限正确
- 调试技巧
  - 使用 -v 标志启用详细日志，查看连接建立过程的详细信息
  - 重点关注 [DEBUG] 级别的日志输出，了解内部处理流程
- 子网相关问题
  - 确认 /112 子网未与其他服务器冲突
  - 检查 ndppd 配置的子网范围是否正确
  - 验证路由表中是否存在正确的 local 路由条目

**更新**：新增子网相关的故障排除指南。
